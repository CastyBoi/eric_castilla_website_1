ETL Project: Data Visualization

This dashboard is the end result of an ETL project I wanted to setup. Has around 5k datapoints, and can be setup to automatically update depending on Data Viz software used.

My process for this was much more streamlined than the dashboards below. I was already familiar with scraping data from websites, but this time I wanted more details (zip codes, transmission, url) and was able to parse JSON data from the AutoTrader Website. (Shoutout AutoTrader for not blocking my IP address like Cars.com). More details on Data Pipeline, Flow, and overall process can be found on my Projects Page.

The end product is a dashboard that shows the Average Price by state, allows easy filtering by Manufacturer, Model, and gives some basic insights on distribution. This endevour was more about learning ETL tools and seting up an automated process vs having a really nice data visualization tool.

Tableau Dashboard: Porsche Data Research

Link: Porsche Research Dashboard

Created a dashboard to display some basic data related to Porsche pricing, mileage, year, and trim.

In order to gather the above information, I familiarized myself with the following: Selenium (Python package), lots of Googling, ChatGPT prompting, more Googling, and lots of trial and error.

The end product is an interative Tableau Public Dashboard, that allows the user to sort through about 2000 data points, select different models, trims, years, etc, and see trends in relation to age and mileage! Also, if you click the images it takes you to Porsche's website and you can build your own 718 or 911.

End Product: utilized several skills, from Python programming, Pandas for data cleaning, regex string identification to separate the information, and Webscraping knowledge. (I even created my own SQL Server to store the information!)




Tableau Dashboard: Manual Car Research

Link: Manual Car Research Dashboard

Created another dashboard to research all manual cars within a 100 mile radius to my zipcode.

The data used in the dashboard above was gathered using a modified Webscraper that also collects the listing links, and more reliably captures and separates trim and model data.

This dashboard is an evolution of the first, and was developed after I had spent more time using Tableau in my day to day work. This one actually holds links to the car listings that were scraped, and are clickable right in the dashboard itself!

End product: Similar outputs and trend recoginition, but with a little more usability and some extra features when compared to the first.

Raspberry Pi Based Data Logger

This project was a long time in the making, as it was quite literally my FIRST endevour into the world of programming and developement. I had an end goal which could be summarized as follows: "Develop a method to log production data, and display the data live using a data visualization software". I had no idea there would be so many moving parts, and was only vaguely familar with many of the concepts involved in this task.

(The clip below is not of the actual data logging device. I didn't feel comfortable sharing actual images since it is being used in production at my company. So I made a simple display with a WRX Hatchback instead.)


Some things I learned over the development, prototyping, and eventual deployment of this project:

  • 3D modeling - CAD Design

  • Python - Linux - Bash Scripts

  • Linux - Bash Scripts

  • Microsoft SQL Server - SQL

  • CRON scheduling - FTP File Transfer

  • Tableau - Data Analytics

  • Pandas - Data Cleaning

I researched, developed, and deployed almost every bit involved in this project. From 3D modeling a bracket to fit the equipment, deploying a SQL Server, writing the code to read and send sensor data, and finally to actually testing and sending the device to be put into production.
